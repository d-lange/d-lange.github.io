CSS Concepts: inline vs. inline-block

As I dive into my second week of Phase: 0 at Dev Bootcamp, I've learned a lot about HTML and CSS.

In this blog post I'm going to explain the difference between displaying an inline element and an inline-block element.

When writing CSS code, we use the property 'display' which helps determine how an element is displayed on the page. From here, we specify whether we want whatever we're styling to be inline, block, or inline-block and with proper syntax looking like this: [display: inline;] or [display: inline-block;].

Visually, when displaying an inline element, it won't look like much, but having multiple elements together, you'll see they are viewed next to one another. This is because an inline element does not affect anything but what's between the tag itself.

A block element allows for modification of other aspects of the element such as the margin or padding from the content inside it, but it forces a new line break after the element, meaning elements can't sit next to one another. This brings us to the final way we can use the 'display' property to style our HTML.

An inline-block element is the best of both worlds. It combines the flexibility of an inline element and the encapsulation of a block element. This means that each element will act as a block, but be allowed to be styled next to one another. This makes styling easier and more organized.
