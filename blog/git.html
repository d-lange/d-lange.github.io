Version Control, Git, and GitHub


January 3rd, 2016


Let's start with Version Control. What is it? Let's simply start by looking at the phrase "Version Control." Those two words imply the controlling of versions. Well, versions of what? In our case, we are talking about organizing software that consists of many versions and modifications. This means, that you might have a version of your software from two weeks ago compared to present time. You might want to go back to check out that code. This is version control. The BENEFITS of version control as a programmer are invaluable. It allows a programmer to revert back to previous work they've done as it is all tracked by using the git commit command. Git takes a snapshot of all work that is committed. It also allows for a lot of experimentation, different versions of your code can be tested on other versions. You're also able to view the history of your code, which can be very beneficial when seeing the progress of the work. It is a very important tool as a Programmer.


Now lets move on to git. What is git? Git is a version control system. Git is what tracks the changes in your code. The way this works is that you go to the master branche of a file you want to work on (git checkout master). To make a change, you make a feature-branch (git checkout -b feature-branch) which allows you to work on the code without it affecting the original piece of code. Once finished with your work and ready to save it. You use a git command called (git add filename) and (git commit -m "commit message"). Git essentially takes a snapshot of everything you committed. Once you commit, you have now saved it to the original piece of code and your history.


Why use GitHub to store our code? That's an interesting question because we already use git. GitHub is essentially a Hub for all your work, but on the internet. You can push your work through git to gitHub for your code and files to live on the internet. It also allows for incredible collaboration globally. It allows you to look at other people's code, learn from it, play with it, tweak it, and vice-versa.
