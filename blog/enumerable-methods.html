Ruby: Enumerable Methods


February 2nd, 2016


Let’s start with the obvious. What is an enumerable? An enumerable is a module that iterates over a collection. In other words, it can search, sort, and manipulate collections of data. There's nothing new conceptually compared to loops, arrays, and hashes, but enumerables make code shorter and most would say nicer to read.


Today I’m going to dissect the enumerable method, .map. The .map method accepts a block of code and affects each element in the collection that .map is invoking. Reading this back to myself, I realize it sounds like I’m describing the .each method. The key difference here is that the .map method returns that transformed collection. More specifically, the .map method returns a copy of a collection without affecting the original copy. This is known as a non-destructive method.


Here’s an example of what the .map method can do:


guitar_brands = [“gibson”, “fender”, “martin”, “taylor”, “gretsch”].map { |guitar| guitar.capitalize}


==> [“GIBSON”, “FENDER”, “MARTIN”, “TAYLOR”, “GRETSCH”]


In the example above you can see that I was given the array, or a list, that includes some colors, the names of various guitar manufacturers. By calling the .map method I iterate it to look through the array and capitalize each element in the array.


There are so many resources online that go into enumerables and the methods one can use. You can also go to the Ruby Docs to find out all of the enumerables that exist.
