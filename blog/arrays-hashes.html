Ruby: Arrays and Hashes

January 25th, 2016

In this blog post I'm going to explain the difference between arrays and hashes in Ruby.

In the early stages of learning Ruby we learn about arrays and hashes and notice they both collect, store and retrieve or lookup information(data). But, what are the real differences in the way they do this?

Lets first look at an array. I think the best way to understand the difference is to think of an array like an ordered list of some kind. Each piece of information in an array represents the number it is within the list. The first piece of information being zero, the second, one, and so on.

If you have a chore list each chore would have a number, or an index, assigned to it. An example would be you'd have a chore list array: chore_list = ["Laundry", "Vacuum", "Garbage"]. "Laundry" represents 0, "Vacuum" is 1 and "Garbage" is 2. If we wanted to lookup a certain chore in our chore list array, we'd simply write chore_list[1] which would return "Vacuum".

A hash works similarly, but is slightly more complex in the way it stores information. A hash has a collection of data, but rather think of it as pairs of information. Each pair is represented by a key and a value. Each key must be unique and you'd use the key to retrieve the value that is paired to it.

Looking at the example of an array, we'd have a hash: chore_list = { "Laundry" => "Sheets", "Vacuum" => "Kitchen", "Garbage" => "Bathroom" }

In this example, the chore would be the key and more specifically what that chore involves is the value. You'd look up information within your hash by identifying the key: chore_list["Laundry"], which ould return "Sheets".
