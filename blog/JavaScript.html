Looping: Ruby vs. JavaScript

In this post I'm going to discuss looping in the context of how it works in programing, specifically the differences between looping in Ruby and JavaScript.

Let's try to understand what looping means. The first word that comes to mind is: repeat. This is actually a pretty good way to describe looping. In programming, looping repeats a certain task through data that the programmer decides to 'loop' through.

In the programming language: Ruby, programmers use a method called .each. This method helps iterate through data structures. For the .each method to work, it has to iterate through a set of data. In programming we refer to a list, such as an array or a hash, which store data.

This is how the the .each method would be implemented in Ruby using an array that is storing the data:

array = ["David", "Lindsay", "Rachel"]

array.each {|i| puts i}

The end result of this code will print out to the console:

"David"

"Lindsay"

"Rachel"

What the .each method is doing here is saying to look at each element inside the array and for every element, print it out to the console. The example uses "puts" rather than "print" or "p" because the array has strings as the data it's storing.

Now let's look at how looping works in JavaScript. Unfortunately, the .each method/function does not exist in the JavaScript language, but the language still has a way to loop through and iterate data structures. JavaScript uses something called the 'for' loop. A 'for' loop is fairly similar to the .each method but the way it is written is a little more involved compositionally.

This is how the 'for' loop would be implemented in JavaScript using the same array we did before:

array = ["David", "Lindsay", "Rachel"]

for (var i=0; i < array.length; i++) { console.log(array[i]); }

The 'for' loop is working using the three parameters set: the variable establishing a counter set to a number, 0 in this case (the beginning of the array), the boolean to establish as long as the counter is less than the length of the array, and to increment the counter each time to look at each element inside the array. Inside the loop we have a simple code block that just tells the computer to print out the array using the counter variable in the for loop, meaning it will print out each element inside the array one at a time.
